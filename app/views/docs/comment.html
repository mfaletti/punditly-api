GET /comments/:id

http://api.punditly.com/1/comments/1234

Returns a single comment, specified by the id parameter.

Parameters
id required The numerical id of the comment
trim_user optional When set to true, each comment returned will include a user object including only the comment author's numerical ID. Omit this parameter to receive the complete user object.

Example Request

GET http://api.punditly.com/1/comments/1234?trim_user=true
Response

{
  "response": {
    "status": 200
  },
  "data": {
    "id": "52ddb9ccb8da75d450000001",
    "text": "goal",
    "event_id": "1",
    "share_count": 0,
    "favorite_count": 0,
    "created_at": "2014-01-21T00:05:32.153Z",
    "user": {
      "id": "52daf3421f31a7e954000001"
    }
  }
}

POST /comments/create

http://api.punditly.com/1/comments/create

Creates a new commentary comment

Parameters
text required The comment to create.
in_reply_to_comment_id optional The ID of a existing comment that this comment replies to
trim_user optional When set to true, the comment returned will include a user object including only the comment author's ID

Example Request

POST http://api.punditly.com/1/comments/create?trim_user=true
POST Data text=Chelsea beat Man City 1-0 at the Etihad
Response

    "status": 200

GET /comments/home_timeline

http://api.punditly.com/1/comments/home_timeline

Returns the most recent comments posted to the authenticating user's home stream, which is made up of the users and team symbols they follow

Parameters
limit optional Specifies the number of results to retrieve. Must be less than or equal to 50. Defaults to 20
since_id optional Returns results with an ID greater than (more recent than) the specified ID
